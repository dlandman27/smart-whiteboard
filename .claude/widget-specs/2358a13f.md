# Sports Widgets — Implementation Spec

**One-liner:** Implement the visual redesign of `SportsWidget.tsx` per spec `e1a16c41` — fixing broken logos, horizontal overflow, missing game grouping, and hardcoded colors — without touching the data layer.

---

## Context

The prior spec (`e1a16c41.md`) defined *what* to build. This spec translates that into *how* to build it, with precise file targets, component contracts, and a clear implementation checklist for the Lead Engineer and devs.

**Affected files (only these should change):**
- `src/components/widgets/SportsWidget.tsx` — full rewrite of `GameRow`, `SportsShell`
- `src/components/widgets/SportsSettings.tsx` — no functional changes; CSS-var audit only
- `src/components/widgets/registry.tsx` — no changes (sizes stay 340 × 420)
- `src/hooks/useSports.ts` — **no changes**
- `server/routes/sports.ts` — **no changes**

---

## User Stories

1. **As a user**, I see team logos (or a clean abbreviation badge fallback) so I can identify teams instantly without reading text.
2. **As a user**, I see games grouped into **Live**, **Upcoming**, and **Final** sections with clear headers so I know what's happening now vs. later.
3. **As a user**, I can resize the widget freely (min 240 px wide) and the layout scales cleanly — no horizontal scrollbar, no clipped text.
4. **As a user**, my favorite team's game is pinned to the top of its group and visually highlighted so I never have to hunt for it.

---

## Acceptance Criteria

### 1. Logo / Fallback Badge
- [ ] Every `<img>` for a team logo has `crossOrigin="anonymous"` and an `onError` handler that swaps to a fallback badge.
- [ ] Fallback badge: circular `<span>` showing team abbreviation (≤ 3 chars), background `var(--wt-surface-raised)`, text `var(--wt-text-muted)`, no broken-image icon ever shown.
- [ ] Logo size: `clamp(18px, 6cqw, 32px)` (or equivalent `ResizeObserver`-driven value).
- [ ] Logo `<img>` has `alt="{team.abbr} logo"`; fallback badge has `aria-label="{team.abbr}"`.

### 2. Game Grouping
- [ ] Games are rendered in three labeled sections in order: **Live** → **Upcoming** → **Final**.
- [ ] Sections with zero games are hidden entirely (no empty header rendered).
- [ ] Section header style: `var(--wt-text-muted)`, `text-transform: uppercase`, `font-size: clamp(9px, 2.5cqw, 11px)`, `letter-spacing: 0.08em`.
- [ ] Live section header includes a pulsing dot using `var(--wt-danger)` (fallback `#ef4444`) and the existing `pulse` CSS animation.
- [ ] Favorite team's game is sorted to the top of whichever section it belongs to.

### 3. Overflow / Responsive Layout
- [ ] No horizontal scrollbar at any widget width ≥ 240 px.
- [ ] No vertical scrollbar at default size (340 × 420) when ≤ 8 games are present. Vertical scroll is allowed when game count genuinely exceeds container height.
- [ ] `SportsShell` uses `ResizeObserver` (same pattern as `WeatherWidget`) to derive a `containerWidth` value; font sizes and padding scale from it.
- [ ] At `containerWidth < 280`: hide win/loss records; show only abbr + score.
- [ ] At `containerWidth ≥ 280`: show records on pre-game rows.
- [ ] All text nodes use `overflow: hidden; text-overflow: ellipsis; white-space: nowrap` — no wrapping.

### 4. Colors & Theming
- [ ] Zero hardcoded color values anywhere in `SportsWidget.tsx` or `SportsSettings.tsx`.
- [ ] Allowed CSS vars: `var(--wt-accent)`, `var(--wt-text)`, `var(--wt-text-muted)`, `var(--wt-border)`, `var(--wt-surface)`, `var(--wt-surface-raised)`, `var(--wt-danger)`.
- [ ] Favorite team highlight: `color-mix(in srgb, var(--wt-accent) 10%, transparent)` background + `2px solid var(--wt-accent)` left border (existing pattern, just de-hardcode).
- [ ] Live score text color: `var(--wt-danger)`.

### 5. Icons
- [ ] All non-logo icons (live dot, any chevrons) use Phosphor icons via the existing `<Icon>` component — no raw SVG or emoji.

### 6. Accessibility
- [ ] Game list container: `role="list"`.
- [ ] Each `GameRow`: `role="listitem"`.
- [ ] Live score/status text: wrapped in `<span aria-live="polite">`.
- [ ] Section headers: `role="heading" aria-level={3}`.

### 7. Component Contract
- [ ] `SportsShell` accepts `{ league: League; settings: SportsWidgetSettings }` — unchanged signature.
- [ ] `GameRow` accepts `{ game: Game; favoriteTeam: string; containerWidth: number }` — `containerWidth` added to drive responsive hiding.
- [ ] `NFLWidget` and `NBAWidget` export signatures unchanged.
- [ ] `SportsSettings`, `NFLSettings`, `NBASettings` export signatures unchanged.

### 8. Registry
- [ ] `registry.tsx` is **not modified** — default sizes remain `{ width: 340, height: 420 }` for both NFL and NBA.

### 9. TypeScript
- [ ] `npx tsc --noEmit` passes with zero errors after changes.
- [ ] No `any` types introduced in `SportsWidget.tsx`.

### 10. No Data Layer Changes
- [ ] `useSports.ts` is **not modified**.
- [ ] `server/routes/sports.ts` is **not modified**.

## Default Size
- **Width:** 340 px
- **Height:** 420 px
*(unchanged — must look correct at this size with no scroll for a typical game day)*

## Settings / Preferences
- **Favorite Team** (existing, string) — team abbreviation; game pinned to top of its section and highlighted.
- No new settings introduced in this implementation.

## Data Sources
- ESPN Scoreboard API via `/api/sports/:league` (proxied by `server/routes/sports.ts`)
- Polled every 60 s via `useNFLScores` / `useNBAScores` (unchanged)
- Supported leagues in server: `nfl`, `nba`, `premierleague`, `laliga`, `ucl`, `mlb`, `nhl` — only `nfl` and `nba` are wired to widgets in this task.

## Implementation Notes for Lead Engineer

> These are architectural nudges — defer to Lead on final approach.

1. **ResizeObserver pattern** — copy the `useResizeObserver` usage from `WeatherWidget.tsx` verbatim. Attach to the root `div` ref; store `containerWidth` in state; pass down to `GameRow`.
2. **Logo CORS** — `crossOrigin="anonymous"` + `onError` fallback badge is the agreed approach. Server-side logo proxying is **out of scope** for this task (see Open Questions).
3. **`SportsShell` refactor** — keep it in `SportsWidget.tsx` for now; extraction to a `sports/` subdirectory is a separate task (see Open Questions).
4. **`var(--wt-danger)` availability** — confirm this var exists in the theme before using; if not, use `#ef4444` as the only permitted hardcoded color exception.
5. **Pulse animation** — reuse the existing `@keyframes pulse` already defined in the global stylesheet; do not add a new one.

## Open Questions

1. **Logo CORS** — ESPN logo URLs (`a.espncdn.com`) may block cross-origin requests even with `crossOrigin="anonymous"`. If the fallback badge fires on every load, should the server proxy logo images? *(Defer to Lead Engineer — architecture/reliability call.)*
2. **`SportsShell` location** — Keep in `SportsWidget.tsx` or extract to `src/components/widgets/sports/SportsShell.tsx` in anticipation of MLB/NHL? *(Defer to Lead Engineer.)*
3. **`var(--wt-danger)` token** — Does this CSS var exist in the current theme? If not, what is the correct token for red/error states? *(Lead or Dev 2 to confirm before implementation.)*
4. **Vertical scroll at short heights** — If user resizes widget to a very short height, show a "X more games" collapsed state or just allow scroll? *(Default: allow scroll, no collapse — confirm with founder if needed.)*
5. **`containerWidth` prop vs. CSS container queries** — `ResizeObserver` state is the established pattern (WeatherWidget), but CSS container queries (`cqw`) would be cleaner. Use `ResizeObserver` for consistency unless Lead prefers container queries. *(Defer to Lead.)*
